============================================================
AUTOMATION GROUP — Website + Admin Portal
Setup & Usage Guide
============================================================

HOW IT WORKS
────────────
This folder is the public static website. Editable content (hero text,
service/solution cards, stats, contact details, etc.) is fetched at page
load from the Admin Portal API (see ../Admin_Portal, a separate Next.js +
Prisma application with its own database and authentication). If the API
is unreachable, each page falls back to the static HTML already in the
file — nothing on the public site ever goes blank.

NOTE: js/cms.js's AUTOMATION_CMS localStorage system is a minimal local
fallback only — the Admin Portal is the single source of truth for
editable content. AUTOMATION_CMS.load()/.save() are used internally as a
local fallback for storing contact-form messages when Supabase isn't
configured (see js/messages.js).

ADMIN PORTAL ACCESS
────────────────────
admin-portal.html is a redirect stub pointing at the deployed Admin Portal
URL. There are no credentials in this repository — admin accounts are
created via the Admin Portal's own setup/seed process (see
../Admin_Portal/prisma/seed.ts, which requires SEED_ADMIN_EMAIL and
SEED_ADMIN_PASSWORD environment variables). Set real credentials there,
not in this folder.

EDITING CONTENT
────────────────
1. Log in to the deployed Admin Portal.
2. Open a page section (Home, About, Services, Solutions, Contact).
3. Edit the fields and save.
4. The corresponding public page picks up the change on next load.

RECEIVING CONTACT FORM MESSAGES
─────────────────────────────────
Messages submitted via contact.html are saved to Supabase (if configured
in js/messages.js) and also forwarded to the Admin Portal's contact inbox
API. Check the Admin Portal dashboard to view and manage enquiries.

DEPLOYMENT
────────────────
This site deploys as a Cloudflare Worker/Pages static asset bundle (see
wrangler.jsonc, worker.js). Update js/cms-loader.js's `data-api` attribute
and js/script.js's map marker / contact details if the Admin Portal moves
to a new domain.
