Overview — What You Are Setting Up

This guide is for the platform administrator only Regular users do not need to do any of this. Once these keys are configured, all users can connect their X accounts through the simple OAuth flow in the app.
X Developer Access Requires Approval Unlike Facebook and LinkedIn, X (Twitter) requires you to apply for developer access and wait for approval before you can create an app. This process can take from a few hours to a few days. The Basic tier (free) is sufficient for this platform.

The two keys you need are:

# These go in your .env.local file and Vercel Environment Variables
TWITTER_CLIENT_ID=your_client_id_here
TWITTER_CLIENT_SECRET=your_client_secret_here

1. Apply for X Developer Access

🐦 Developer Portal Registration

1

Go to the X Developer Portal

Open your browser and navigate to https://developer.twitter.com/en/portal/dashboard. This is X's official developer portal where all developer apps are managed. Make sure you are logged in to the X account you want to use as the developer account. This should be your main business X account.

2

Sign up for a developer account

If you have never created an X developer account before, you will be prompted to apply. Click "Sign Up" or "Apply for Access". X will ask you to describe how you plan to use the developer tools. Be honest and specific — this helps with approval. Write something like: "I am building a social media management platform for my business that allows scheduling and publishing posts to X on behalf of authorized users."

3

Select the "Basic" access tier

X offers several access tiers. For this platform, the Basic tier (which is free or very low cost) is sufficient. It provides enough API calls for regular posting. Select Basic and continue. If prompted for payment, the Basic tier cost is minimal and required for API posting access as of 2024.

X changed its API access policies in 2023–2024. A paid Basic tier is now required to write (post) tweets via the API. The Free tier only allows reading tweets. Make sure you select Basic or higher to enable posting functionality.
4

Complete your developer profile

Fill in all required fields including your use case description, your country, and agree to X's Developer Agreement and Policy. Click Submit. X will review your application. You will receive an email when your developer account is approved — this usually takes a few hours but can take up to 48 hours.

2. Create a Project and App

🐦 Project & App Creation

1

Open the Developer Portal Dashboard

After your developer account is approved, go to https://developer.twitter.com/en/portal/dashboard. You will see the developer dashboard with a left sidebar showing Projects and Apps. X organizes things into Projects (containers) which contain Apps (the actual API credentials).

2

Create a new Project

In the left sidebar, click "Projects & Apps" → "Overview". Click the "+ New Project" button. A setup wizard will open. Enter a project name such as Apex Social Automations. Select a use case — choose "Making a product for other businesses (Automating and bot accounts)" or the most relevant option. Add a project description explaining the platform. Click Next.

3

Create an App inside the Project

The wizard will immediately ask you to create an App inside your new project. Enter your app name — use something like Apex Social Automations App. Select the environment — choose Production (not Development) since this is a real deployed application. Click Next.

4

Save the keys shown — IMPORTANT

After creating the app, X will show you your API Key, API Key Secret, and Bearer Token one time only. Copy all three and save them immediately in a secure location. However, for this platform we use OAuth 2.0 (not OAuth 1.1), so the keys you will primarily need are the Client ID and Client Secret, which you will find in the next section. Still save all the keys shown — you may need them for troubleshooting.

X only shows these API keys ONCE. If you close the page without saving them, you will have to regenerate them. Click the copy icons for each key and paste them into a secure document immediately.

3. Get Your OAuth 2.0 Client ID and Client Secret

🐦 Copying the Keys

1

Go to your App Settings

In the left sidebar, under Projects & Apps, click on the name of the app you just created. This opens the app's detail page. Click the "Keys and Tokens" tab at the top.

2

Find the OAuth 2.0 Client ID and Client Secret

Scroll down the Keys and Tokens page until you find the section labelled "OAuth 2.0 Client ID and Client Secret". Click "Generate" if they have not been created yet, or "Regenerate" if you need new ones. A popup will appear showing your Client ID and Client Secret. Copy both immediately.

  • Client ID → this is your TWITTER_CLIENT_ID
  • Client Secret → this is your TWITTER_CLIENT_SECRET
Example: TWITTER_CLIENT_ID = dGhpczEyM2FiY2Q0NTY3ODk
Just like the API keys, the Client Secret is only shown ONCE. Copy it immediately and store it securely. If you lose it, you will need to regenerate it and update all your environment variables.

4. Enable OAuth 2.0 on Your App

🐦 OAuth 2.0 Configuration

1

Go to App Settings

In the left sidebar, find your app under Projects & Apps and click on it. Then click the "Settings" tab at the top. Scroll down to find the "User authentication settings" section. Click "Set up" or "Edit".

2

Enable OAuth 2.0

In the User Authentication Settings page, toggle OAuth 2.0 to ON. Make sure OAuth 1.0a is also available but OAuth 2.0 is what this platform uses for the authorization flow.

3

Set the App Type

Under the OAuth 2.0 section, find the Type of App field. Select "Web App, Automated App or Bot". This tells X that your app runs on a server and handles OAuth flows on behalf of users.

5. Set the Callback URL

🐦 Callback & Website Configuration

1

Add the Callback URI

Still on the User Authentication Settings page, find the field labelled "Callback URI / Redirect URL". Click Add and enter the following URL exactly:

https://app.apexrmgroup.com/api/social/twitter/callback
For local development, also add: http://localhost:3000/api/social/twitter/callback
2

Add the Website URL

In the "Website URL" field, enter your main website URL:

https://app.apexrmgroup.com
3

Save the settings

Click Save at the bottom of the User Authentication Settings page. X will confirm the changes. After saving, you will see a summary of your OAuth settings on the app's Settings page.

6. Set App Permissions

🐦 Read & Write Permissions

1

Go to App Permissions

Click the "Settings" tab of your app and find the "App permissions" section. This controls what level of access your app has to users' X accounts.

2

Set permissions to "Read and Write"

Click Edit next to App Permissions. Select "Read and Write". This is required so that the app can post tweets on behalf of users. "Read only" will not allow publishing. Click Save to confirm the change.

After changing permissions, you must regenerate your Access Token and Secret (if you use OAuth 1.1) for the change to take effect. For OAuth 2.0 (which this app uses), the scope is requested at authorization time using the URL parameters, so users connecting their accounts after this change will automatically have the correct permissions.
3

Confirm the OAuth 2.0 scopes

The app requests the following OAuth 2.0 scopes from users when they connect their X account:

  • tweet.write — Allows creating new tweets on behalf of the user
  • tweet.read — Allows reading tweets and user timeline
  • users.read — Allows reading basic user profile information
  • offline.access — Allows the app to refresh the access token without requiring the user to re-authorize

These scopes are built into the app's code and will be requested automatically when users go through the OAuth flow.

7. Add Your Keys to the App

You now have everything you need You have your TWITTER_CLIENT_ID and TWITTER_CLIENT_SECRET. Add them in two places.

🐦 .env.local (Local Development)

1

Update your .env.local file

Open the file .env.local in your project folder and update the Twitter/X lines:

TWITTER_CLIENT_ID=dGhpczEyM2FiY2Q0NTY3ODk
TWITTER_CLIENT_SECRET=abcdefghij1234567890klmnopqrst

🐦 Vercel Environment Variables (Production)

1

Add to Vercel

Log in to vercel.com → open your project apex-social-automations → click Settings → Environment Variables. Find or add TWITTER_CLIENT_ID and TWITTER_CLIENT_SECRET with the values you copied. Select All Environments for each. Click Save. Then redeploy your project from the Deployments tab for the changes to take effect.
